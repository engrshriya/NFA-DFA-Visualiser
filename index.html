NFA

Non-Deterministic

From one state, you can go to multiple next states on the same input — or even move without reading input (ε moves).

🌀
DFA

Deterministic

From one state, exactly one next state exists for each input. Predictable, clean, and easy to simulate.

🎯

Key Ideas Before We Start

01

States (Q)

Circles in the diagram. Each represents a configuration the machine can be in.

02

Alphabet (Σ)

The set of input symbols the machine reads, like {0, 1} or {a, b}.

03

Start State (q₀)

Where the machine begins. Shown with an arrow pointing to it.

04

Accept States (F)

Double-circled states. If we end here after reading input, the string is accepted.

05

Transition (δ)

Rules that say: "In state X, on input a, go to state Y." Arrows in the diagram.

06

Subset Construction

The core algorithm: each DFA state represents a set of NFA states. That's the magic!
